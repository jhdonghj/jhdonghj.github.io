Particle Filtering

Sometimes the state space is too large to run the exact inference. We need to approximate inference.

  • Track samples of X, not all values
  • Samples are called particles
  • Time per step is linear in the number of samples

Now that we’ve seen how to recover a belief distribution from a particle list, all that remains to be discussed is how to generate such a list for a timestep of our choosing.

Particle Filtering Simulation

Particle filtering simulation begins with particle initialization. And then the simulation takes a time elapse update followed by an ovservation update at each timestamp.

  • Time Elapse Update: Update the value of each particle according to the transition model. That is $t_i\gets \text{sample}(P(T_{i+1}|t_i))$
  • Observation Update: Use the sensor model $P(F_i|T_i)$ to weight each particle according to the probability dictated by the observed evidence and the particle’s state. The algorithm of observation update is as follows:
    • Calculate the weights of all particles according to $P(f_i|t_i)$ where the sensor reading is $f_i$.
    • Calculate the total weight for each state.
    • If the sum of all weights across all states is 0, reinitialize all particles.
    • Else, normalize the distribution of total weights over states and resample your list of particles from this distribution.

Summary

  • Markov models encode time-dependent random variables with the Markov property. We use probability inference to compute the belief distribution.
  • Hidden Markov Models are Markov models with hidden states. We compute the belief distribution by observing evidence.

Sometimes, running exact inference on these models can be too computationally expensive, in which case we can use particle filtering as a method of approximate inference.
